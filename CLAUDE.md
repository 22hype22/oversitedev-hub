# CLAUDE.md — Oversite

Guidance for AI agents (Claude Code, Cursor, etc.) working in this repo.

## Two deployable units

1. **Dashboard** (this directory) — React 18 + Vite 5 + Tailwind v3 + TypeScript 5.
   Backend = Lovable Cloud (managed Supabase). Edge functions in `supabase/functions/`.
2. **Worker** (`/worker`) — Standalone Node.js 20+ process. Has its own `package.json`,
   `Dockerfile`, and `railway.json`. See [`worker/CLAUDE.md`](./worker/CLAUDE.md).

**These are separate npm projects.** Do not import from one into the other.

## Hard rules (dashboard)

- Never edit `src/integrations/supabase/client.ts` or `types.ts` — auto-generated.
- Never edit `.env` — auto-generated by Lovable Cloud.
- Never store secrets in client code. Use edge functions + Cloud secrets.
- Roles live in `user_roles` (never on `profiles`). Use the `has_role()` SECURITY DEFINER fn.
- All colors via semantic tokens in `index.css` / `tailwind.config.ts`. No raw hex/`text-white`.
- Use `@/` import alias.

## Hard rules (worker)

See [`worker/CLAUDE.md`](./worker/CLAUDE.md).

## Where things live

- `src/pages/` — top-level routes
- `src/components/dashboard/` — authenticated dashboard UI
- `src/components/admin/` — admin-only UI (gated by `has_role`)
- `src/hooks/` — data hooks (`useBotOrders`, `useBotLogs`, `useBotNotifications`, …)
- `supabase/functions/` — edge functions (auto-deployed)
- `supabase/migrations/` — DB migrations (apply via the migration tool, not by hand)
- `worker/src/addons/` — Discord addon implementations

## Common tasks

- **Add a Discord feature** → new addon in `worker/src/addons/`, register in
  `worker/src/addons/index.ts`, surface a builder in `src/components/dashboard/`.
- **Add an admin tool** → component in `src/components/admin/`, mount in `src/pages/Admin.tsx`,
  gate by `has_role('admin')`.
- **Schema change** → write a migration via the migration tool. Never hand-edit `types.ts`.
